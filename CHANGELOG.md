## 변경 이력 작성 가이드

새로운 변경사항이 있을 때는 다음 형식을 따라 작성해주세요:

1. `docs/changelog/YYYY-MM-DD.md` 파일 생성
2. 이 파일(`CHANGELOG.md`)에 날짜별 항목 추가
3. 상세 내용은 날짜별 파일에 작성

### 파일 명명 규칙
- 형식: `YYYY-MM-DD.md` (예: `2026-02-18.md`)
- 위치: `docs/changelog/` 폴더

### 작성 내용
- 변경 날짜
- 수정된 파일 목록
- 변경 내용 상세 설명
- 주요 함수/기능 추가/수정 사항
- UI/UX 개선사항
- 기술 스택 변경사항
- 참고 파일
